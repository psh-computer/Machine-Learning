# Introduction
이 프로젝트는 삼성전자의 역사적 주가 데이터를 바탕으로 미래의 가격을 예측하는 시계열 예측 모델을 구축한다. 비선형적이고 변동성이 큰 금융 데이터를 처리하기 위해 LSTM 모델을 사용하였다.

# Data Engineering
 ### - MinMaxScaling
 주가 데이터는 단위가 크고 변동 폭이 일정하지 않아 학습의 수렴 속도를 늦춘다. 따라서 데이터를 [0,1] 범위로 정규화하였다.

 ### - Sliding Window
 60일간의 데이터를 하나의 구간으로 묶고 다음 날의 종가를 정답으로 설정했다. 이는 모델이 추세와 변동성을 학습하게 한다.

 # Model Architecture
 ### - LSTM
일반 RNN은 정보가 전달될수록 앞선 정보가 소실되는 기울기 소멸 문제가 발생한다. 하지만 LSTM은 Cell State를 통해 60일 전의 중요한 시장 상태를 소멸하지 않고 보존하도록 설계된다.

### - Hidden State 초기화
매 배치마다 h0, c0 를 0으로 초기화하여 이전 배치의 영향이 현재 학습에 간섭하지 않도록 독립성을 부여했다.

### - Many-to-One 구조
60개의 타임스텝 출력을 모두 사용하지 않고, 마지막 타임스텝(out[:,-1,:])의 정보만을 추출하여 FC 레이어에 전달했다. 이는 60일간의 모든 정보를 집약한 요약본으로 결론을 내리게 한다.

# Technical Challenges
### - Batch First 설정
데이터 연산의 직관성을 위해 (Batch, Time_Step, Feature) 구조로 바꾼다.

### - MPS 가속화 에러 해결
MacBook 환경에서 GPU(MPS) 연산을 수행할 때, 입력 데이터와 초기 상태 텐서(h0, c0)가 서로 다른 장치에 머물러 발생하는 RuntimeError를 .to(device)로 장치 일치화를 수행하여 해결했다.

# Analysis & Limitations
### Lagging
모델의 예측 그래프가 실제 주가보다 한 스탭 뒤처져 나타는 경향이 있다. 이는 모델이 주가의 근본적인 원리를 이해하기보다 그 이전의 값에 의존하는 경향이 강하다는 뜻이다.
성능 개선을 위해 단순 가격 정보 외에, 거래량, 이동평균, 혹은 뉴스 텍스트의 감성 분석 데이터를 다변량으로 입력하는 구조로 확장해야한다.
